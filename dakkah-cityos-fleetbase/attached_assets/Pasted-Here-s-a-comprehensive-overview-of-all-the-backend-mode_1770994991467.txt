Key Gaps Summary
Fleet Maintenance (Maintenance, WorkOrder, Part, Warranty) — Models exist but UI is basic
Telematics/IoT (Sensor, Telematic, Device) — Framework exists, needs external service integration
Reporting (Report, ReportExecution) — Backend models exist, UI is limited
OutboxEvent / IntegrationLog — No dedicated admin UI for viewing these logs
WorkflowRegistry — Depends on Temporal Cloud connection which isn't active
LoginAttempt — Security audit model with no admin visibility
FoodTruck — Niche storefront feature, partially implemented
Reviews/Votes — Basic implementation in storefront
Overall: ~127 unique models, ~90% have API endpoints, ~80% have admin panel UI. The main gaps are in fleet maintenance depth, IoT/telematics integration, and some internal audit/logging visibility.
