Implementation Plan
Phase 1: Middleware Injection (Foundation)
Goal: Make NodeContext available on every request across all modules. Zero impact on existing behavior — purely additive.

Files to create/modify (all in api/packages/cityos/src/):

File	Action	Purpose
Providers/CityOSServiceProvider.php	Modify	Push 3 middleware into fleetbase.protected and storefront.api groups
What it does: Every FleetOps, Pallet, Storefront, and Core request now has $request->attributes->get('node_context') available. Modules that don't check it are completely unaffected. Governance enforcement (residency + classification) applies globally.

Phase 2: Database Migrations (Add columns)
Goal: Add tenant_uuid, country_code, city_uuid, sector_uuid columns to key tables in other modules so data can be stamped and filtered.

Migration files to create (in api/packages/cityos/migrations/):

Migration	Tables affected
add_node_context_to_fleetops	orders, drivers, vehicles, fleets, places, service_areas
add_node_context_to_pallet	pallet_warehouses, pallet_inventories, pallet_products, pallet_stock_transactions
add_node_context_to_storefront	storefront_stores, storefront_products, storefront_checkouts
All columns are nullable so existing data isn't affected. No schema breaks.

Phase 3: Model Observers (Stamp on write)
Goal: When a record is created or updated with NodeContext present, automatically stamp the tenant/country/city/sector onto the record.

Files to create:

File	Observes	Stamps
Observers/FleetOpsNodeContextObserver.php	Order, Driver, Vehicle, Fleet, Place	tenant_uuid, country_code, city_uuid, sector_uuid from NodeContext
Observers/PalletNodeContextObserver.php	Warehouse, Inventory, Product, StockTransaction	Same fields
Observers/StorefrontNodeContextObserver.php	Store, Product, Checkout	Same fields
Observers/OutboxLoggingObserver.php	All key models	Logs create/update events to cityos_outbox
Registration: All observers registered in CityOSServiceProvider::$observers array — the same pattern FleetOps and Storefront already use.

Graceful degradation: If no NodeContext is on the request (no headers sent), the observer simply skips stamping. Existing behavior preserved.

Phase 4: Global Query Scopes (Filter on read)
Goal: When NodeContext is present, automatically filter queries to only return data belonging to the resolved tenant.

Files to create:

File	Purpose
Scopes/NodeContextScope.php	Reusable global scope that filters by tenant_uuid when NodeContext is present
Support/NodeContextScopeRegistrar.php	Registers the scope on target models during boot
Logic: If node_context is on the request AND has a resolved tenant, add WHERE tenant_uuid = ? to queries. If not present, no filter (backward compatible). Uses company_uuid as fallback when tenant_uuid isn't populated yet.
